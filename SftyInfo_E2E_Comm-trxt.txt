CRC calculation
Cyclic redundancy check (CRC): An 8-bit CRC is explicitly sent with polynomial 0x1D with an initial value 0x00 and a final XOR-value 0x00.
E2E Profile 1 shall use CRC-8-SAE J1850 for CRC calculation. It shall use 0x00 as the start value and XOR value.
The CRC is calculated over the low byte of the DataID and the data itself starting from the byte B1.
textLow byte of DataID → [CRC] [Counter] [Sig1] [0xFF] [Sig1]

SftyInfo CAN message layout:
The DataID of SftyInfo is 0x8BF (LowByte: 0xBF, HighByte: 0x08)
Signal layout (example):

SftyInfo_Cls: Start 0, Length 8
SftyInfo_Alvctr: Start 8, Length 4
SftyInfo_DIDNibble: Start 12, Length 4
SftyOK: Start 16, Length 2
CrashDetnNeg: Start 18, Length 2


Example of CRC calculation: SftyInfo (VIC) (2/2)

The HighByte of DID shall be 0 and the unused bits shall be 1.
The signals SftyOK and CrashDetnNeg are set to „1“.

To calculate the CRC use the function Crc_CalculateCRC8() (see python example) with an initial value 0x00 and a final XOR-value 0x00.
The lookup table of Crc-8 with polynom 0x1D is used.
The calculation of the data 0xBF 0x00 0x8E 0xF5 using CRC-8-SAE J1850 gives 0x02.

CRC
CRC-8-SAE J1850 – 0x1D (x⁸ + x⁴ + x³ + x² + 1), but with different start and XOR values (both 0x00).
This CRC is provided by CRC library. Starting with AUTOSAR R4.0, the SAE8 CRC function of the CRC library uses 0xFF as start value and XOR value. To compensate, the E2E Library applies additional XOR 0xFF operations.
Note: This CRC polynomial is different from the CRC-polynomials used by FlexRay, CAN and LIN.
